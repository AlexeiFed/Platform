# Оплата: ЮMoney (основной сценарий) и опционально Яндекс.Формы

Техническая спецификация для переноса в другое приложение: [`payment-implementation.md`](./payment-implementation.md).

Платформа умеет два варианта подтверждения оплаты:

1. **Рекомендуемый:** прямой перевод через **QuickPay ЮMoney** со страницы сайта + официальное **HTTP-уведомление ЮMoney** на `POST /api/payments/yoomoney-notification` (подпись `sha1_hash`). Подтверждение приходит **от ЮMoney**, а не от Яндекс.Форм.
2. **Запасной:** внешняя ссылка (например Яндекс.Формы) + `POST /api/payments/form-webhook` с заголовком `x-platform-payment-secret` и JSON.

Если на сервере задан **`YOOMONEY_WALLET_RECEIVER`**, при нажатии «Перейти к оплате» открывается **наша** страница `/catalog/<slug>/oplatit`, затем форма на `yoomoney.ru/quickpay/confirm` с полем **`label`** = внутренний номер заказа (`paymentRef`). ЮMoney после зачисления шлёт POST на ваш домен.

**Важно:** ЮMoney **не присылает** HTTP-уведомление, если пользователь **просто закрыл** страницу оплаты и перевод не произошёл. Поэтому на странице оплаты есть кнопка **«Оплату не завершил»** — она помечает заявку в БД как отменённую (`CANCELLED`). Пока пользователь не нажал её и не прошёл оплату, статус остаётся `PENDING` (= «оплата не завершена»).

---

## 1. Переменные `.env` на сервере

| Переменная | Назначение |
|------------|------------|
| `YOOMONEY_WALLET_RECEIVER` | Номер кошелька получателя (`41001…`), без пробелов. |
| `YOOMONEY_NOTIFICATION_SECRET` | **Секретное слово** из настроек HTTP-уведомлений кошелька ЮMoney (для проверки `sha1_hash`). |
| `PAYMENT_FORM_WEBHOOK_SECRET` | Только если используете вебхук **формы** (`/api/payments/form-webhook`). |

После изменений перезапустите приложение.

Публичный URL для уведомлений ЮMoney (HTTPS, без самоподписанного сертификата в проде):

`https://<ваш-домен>/api/payments/yoomoney-notification`

---

## 2. Настройка кошелька ЮMoney

1. Кошелёк **не анонимный**, с возможностью принимать переводы (по правилам ЮMoney).
2. Включить **HTTP-уведомления**: [настройки уведомлений](https://yoomoney.ru/transfer/myservices/http-notification) — указать URL выше, скопировать **секретное слово** → в `YOOMONEY_NOTIFICATION_SECRET`.
3. Номер кошелька → в `YOOMONEY_WALLET_RECEIVER`.
4. Кнопка **«Проверить»** в кабинете ЮMoney: сервер должен ответить **200 OK** (тестовый запрос).

Документация по формату POST и проверке `sha1_hash`:  
https://yoomoney.ru/docs/payment-buttons/using-api/notifications  

Документация по полям формы QuickPay (`label`, `successURL`, `sum`, …):  
https://yoomoney.ru/docs/payment-buttons/using-api/forms  

---

## 3. Настройка курса в админке LMS

- Цена в рублях **> 0**.
- Для **публикации** платного курса нужно **либо** задать на сервере `YOOMONEY_WALLET_RECEIVER`, **либо** указать «Ссылку на форму оплаты» (запасной сценарий).
- Поле «Ссылка на форму оплаты» при включённом ЮMoney на сервере **не обязательно**.

---

## 4. Поведение для пользователя

1. «Перейти к оплате» → при наличии `YOOMONEY_WALLET_RECEIVER` открывается страница **оплаты на сайте** → «Перейти к оплате в ЮMoney» → сайт ЮMoney.
2. После успешного перевода ЮMoney дергает **`/api/payments/yoomoney-notification`** → в БД платёж `SUCCEEDED`, выдаётся доступ.
3. После оплаты пользователь может попасть на `…/catalog/<slug>?payment=yoomoney_ok` (поле `successURL`) — там подсказка нажать «Проверить доступ».
4. Если оплату **не завершил**: на странице оплаты нажать **«Оплату не завершил»** — заявка `CANCELLED`; иначе она останется `PENDING`, пока не истечёт срок (отдельный cron при желании можно добавить позже).

---

## 5. Запасной вариант: только Яндекс.Формы

Если **`YOOMONEY_WALLET_RECEIVER` не задан**, используется ссылка на форму и вебхук **`/api/payments/form-webhook`** (см. комментарии в коде маршрута). В этом случае подтверждение идёт **не от ЮMoney напрямую**, а от того, кто шлёт POST (часто форма или Make).

---

## 6. Чек-лист (ЮMoney как источник правды)

| ☐ | Действие |
|---|----------|
| ☐ | `YOOMONEY_WALLET_RECEIVER` и `YOOMONEY_NOTIFICATION_SECRET` в `.env` на проде |
| ☐ | В кошельке ЮMoney указан URL `/api/payments/yoomoney-notification` и проверен тест |
| ☐ | Курс опубликован, цена > 0 |
| ☐ | Пользователям объяснить кнопку «Оплату не завершил», если закрыли ЮMoney без перевода |
